Описание БД

В данном приложении используется база данных некоторой социальной сети. Узлами являются люди (пользователи, метка Person) и группы (сообщества по интересам, метка Group). У Person есть следующие атрибуты - name (имя человека, строка), surname (фамилия человека, строка), birth_data (дата рождения человека, например '12.06.2004'). У Group присутствуют следующие атрибуты - name (название сообщества, строка), topic (общая тематика сообщества, строка), number_of_publications (число публикаций, целое число). Между людьми есть связи дружбы и ненависти (FRIENDS и HATES), не имеющие атрибутов. Между людьми и группами есть связь FOLLOWS, имеющая атрибут since, хранящий дату подписи пользователя на группу.

На данном уроке поговорим подробнее про запросы на выборку данных и их структуру:

MATCH <шаблон>

‘WHERE <условие>’

RETURN <выводимые данные>

‘ORDER BY переменная.атрибут ‘ASC | DESC ….’’

‘SKIP n’

‘LIMIT l’;

Далее рассмотрим незнакомые нам составляющие данного запроса подробнее.

Cекция WHERE

Данная секция навешивает ограничение на данные, подходящие под шаблон в секции MATCH. Условие содержит различные операции сравнения и логические операции (NOT, AND, OR, XOR – 'НЕ', 'И', 'ИЛИ', 'ИСКЛЮЧАЮЩЕЕ ИЛИ'). Условием может быть ограничение на метку узла, на значения атрибута узла или отношения, проверкой на существование атрибута, регулярное выражение, шаблон… Приведём пару примеров использования WHERE (попробуйте сами понять, что они возвращают):

  • MATCH (n) WHERE n:Person RETURN n

  • MATCH (n:Person) WHERE n.name = “Oleg” RETURN n

  • MATCH (n:Person) WHERE EXISTS(n.birth_data) RETURN n (проверка на существование атрибута)

  • MATCH (n:Person) WHERE n.birth_data IS NULL RETURN n (так же проверка на существование атрибута (между IS и NULL может стоять NOT))

Использование регулярных выражений и различная фильтрация строк будет рассмотрена в последующих уроках.

Cекция ORDER BY

Следующая секция позволяет сортировать результаты запроса, синтаксис данной секции был представлен выше. Рассмотрим пару нюансов ее использования:

  • Опция DESC означает, что результат запроса будет отсортирован по убыванию. ASC, соответственно отвечает за сортировку по возрастанию.

  • Сортировку можно производить по нескольким атрибутам, старшинство атрибутов определяется слева направо, то есть сначала сортировка произойдет по самому левому атрибуту и далее по порядку.

  • Сортировку можно применить к функции, использованной на значениях атрибутов. В дальнейшем вы больше узнаете о функциях.

Cекция SKIP и LIMIT

Данные секции отвечают за ограничение результата запроса соответственно с начала и конца. То есть, если в запросе присутствует следующий текст:

SKIP n LIMIT l

А всего запрос должен был вывести m значений, то с применением данных ограничений он выведет все значения начиная с n+1 и заканчивая l значением. SKIP пропускает первые n значений, LIMIT же пропускает все значения после l-ого.
